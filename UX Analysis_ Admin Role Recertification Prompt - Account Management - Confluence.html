    UX Analysis: Admin Role Recertification Prompt

    • Created by Bersofsky, Yvonne M. (Contractor), last modified on Oct 31, 2024

    Overview

    As part of ongoing security enhancements, NSF plans to have organization administrators review and verify their users' assigned roles in Research.gov annually. In support of that goal, a new annual prompt and application updates are needed.

    Admin Role Recertification Modal Prompt - To Be

    Description

    Each year, organizations will be required to review and confirm their users' roles. When an admin for the organization signs in, they will see a modal asking them to complete the review process.

    The logic for this will be similar to the Annual Profile Review Trigger. However, different flags and logic are needed. 

    Screenshots

    Content may be updated, but functional requirements will probably not change:

    New data elements

    • Role recertification status Has user role recertification been completed for this organization for the current year? Yes/No
      • If the answer is "No" for any of the admin's registered organizations, then the prompt should be displayed, or the admin should be redirected to complete role review and confirmation.
    • Deferred role recertification count:
      • Admins should only see the modal until the max deferred count has been reached.
        • TBD: Deferred count tentatively to be assigned to each individual admin. 
          • The initial request was to assign the deferred count to the organization.
          • But if the deferred count is assigned to the organization, then an admin could be redirected to complete role recertification without ever seeing the modal, which would be confusing.
      • Each time the admin closes the modal or clicks a button inside the modal, the deferred count is incremented by 1.
      • After the max deferred count is reached, the next time the admin signs in, they should be redirected to the page (TBD) to complete role review and confirmation.

    Additional requirements

    1) Admins who have more than one organization:

    • Admins should only complete this process once per year.
    • The prompt should appear if role recertification needs to be completed for any of the admin's organizations, but the admin should not see multiple prompts for different organizations.
      • TBD: Will the annual prompt be at the same time, once a year, for all organizations, or will it be based on when each organization last completed the review?
      • We should probably use the same approach as the Annual Profile Review Trigger
    • If an admin has already responded to the prompt for one of their organizations, then all of their organizations should also be marked as done.
      • 10/31/2024 question: If we go with option 2 instead of option 1, we should discuss whether the admin dismissing the info message is sufficient to mark that admin's organizations as "Done" or if the message should continue to display to other admins for that org. 
      • When completing this process, admins will review and confirm all of their users for all of their organizations at the same time.

    2) Organizations that have more than one admin:

    • As soon as one admin completes role recertification for an organization, that organization should be marked as done (see Role recertification status above).
    • Scenario 1:
      • Admin A has completed role recertification for Organization X.
      • Organization X's other admin, Admin B, has no other organizations.
      • Admin B should not see the prompt or be redirected to complete role recertification.
    • Scenario 2:
      • Admin A has completed role recertification for Organization X.
      • Admin B is an admin for both Organization X and Organization Y, and role recertification has not been completed for Organization Y.
      • Admin B should see the prompt or be redirected to complete role recertification.

    Admin Role Recertification Screen To Be

    When either of the following conditions is true, the admin will be redirected to a page to complete role recertification:

    1. User clicked "Review my users" in the above modal
    2. Deferred role recertification count has reached the max allowed value

    The location of the redirect page is TBD Admins will either be redirected to a standalone page or to the View My Users page.

    • In either scenario, they will use the Manage User Roles table to complete the review.

    New data elements

    • Role recertification date:
      • Date role recertification was completed for each organization
    • Role recertification completed by: 
      • NSF ID of the admin who completed role recertification for an organization

    High-level workflow

    Wireframes

    Option 1: Role Recertification Prompt Redirects Admins to a Standalone Page


    https://29isy2.axshare.com/#id=m158l8&p=prompt__deferrals_remaining

    https://29isy2.axshare.com/#id=fwf8a1&p=prompt__deferrals_remaining_-_more_than_one_allowe

    • Clicking the "x" or "Remind me later" takes users to the signed in Research.gov home page.
    • Clicking "Review my users" takes users to a standalone page.
    • Any action (click a button or close the modal) increments the deferred role recertification count by 1.
    • Not yet defined: Number of times an admin can defer review.
    • Number of times deferred should be specific to the admin (and not to the organization), so that each admin has an opportunity to see the modal prompt at least one time before being redirected to the review page.

    https://29isy2.axshare.com/#id=zqk2w6&p=review___confirm

    • After clicking "Review My Users" or reaching max deferred role certification count, user is redirect to a standalone page with the Manage User Roles table and a confirmation form
    • Cancel button only displayed if the deferred count has not reached the max value
    • Submit button disabled until user checks the box

    Email notification to org admins (not created yet)

    • All admins of a recertified organization will receive an email notification letting them know:
      • which of their organizations have completed the process
      • date completed
      • admin who confirmed/updated the roles

    https://29isy2.axshare.com/#id=kle8be&p=view_my_users__success_page

    • This is a new success message.
    • It should display at the top of the View My Users page after the user submits the form.

    https://29isy2.axshare.com/#id=w9hdx7&p=prompt__only_1_deferral_allowed_for_each_organizat

    https://29isy2.axshare.com/#id=cr2wfl&p=prompt__more_than_1_deferral_allowed_for_each_orga

    • Clicking the "x" or "Remind me later" takes users to the signed in Research.gov home page.
    • Clicking "Review my users" takes users to the View My Users page.
    • Any action (click a button or close the modal) increments the deferred role recertification count by 1.
    • Not yet defined: Number of times an admin can defer review.
    • Number of times deferred should be specific to the admin (and not to the organization), so that each admin has an opportunity to see the modal prompt at least one time before being redirected to the review page.


    https://29isy2.axshare.com/#id=rlht4p&p=view_my_users__review_required

    • After clicking "Review My Users" or reaching the max recertification deferred count, admins are redirected to the View My Users page
    • A dismissible info message appears at the top of the page and persists until:
      • The user closes the message by clicking on the x, or
      • The user updates a user or user role
      • TBD: Could the message also be dismissed if all of the user's organizations have already been recertified by another admin?
